Premiers pas avec Pyperun

Un pipeline minimal de traitement de données IoT temporelles. De ton fichier CSV brut jusqu'au parquet agrégé prêt pour le ML, l'export PostgreSQL et CSV — en une seule commande.

Python 3 CSV → Parquet 10 traitements CLI + UI + MCP

1

Installer

Le plus simple — un one-liner qui build l'image, démarre une instance (UI + API + MCP + scheduler) et affiche son URL + token :

# Docker — multi-instance, données préservées à chaque mise à jour curl -fsSL https://raw.githubusercontent.com/julienby/pyperun/master/install.sh | bash -s -- my-instance

Ou en local, depuis la racine du projet, en mode éditable :

# installe pyperun + outils de dev pip install -e ".[dev]" # vérifie que la commande répond pyperun --help

💡 Option DuckDB : pip install -e ".[duckdb]" pour activer l'export analytique.

2

Créer un dataset

init échafaude l'arborescence des dossiers et un flow par défaut.

pyperun init MY-EXPERIMENT

Cela crée :

datasets/MY-EXPERIMENT/ ├── 00_raw/ # ← tes CSV bruts vont ici ├── 10_parsed/ ├── 20_clean/ ├── … └── 40_aggregated/ flows/my-experiment.json # ← définition du pipeline

🐳 Sur une instance Docker, flows/ est monté en lecture seule dans le conteneur : ne lance pas pyperun init dedans. Échafaude depuis l'hôte avec le helper pyperun-init (crée le flow + dataset à toi, éditables) :

# installer une fois cp scripts/pyperun-init ~/.local/bin/ && chmod +x ~/.local/bin/pyperun-init # échafauder sur une instance installée pyperun-init my-instance MY-EXPERIMENT
3

Déposer les données brutes

Copie tes fichiers CSV dans datasets/MY-EXPERIMENT/00_raw/. Format attendu : sans header, point-virgule, paires clé:valeur.

2026-01-20T09:07:58.142308Z;m0:10;m1:12;outdoor_temp:18.94
bio_signal
colonnes m0–m11 · entiers (Int64)
environment
outdoor_temp · flottant (Float64)
4

Lancer le pipeline

La commande centrale. Elle exécute toutes les étapes dans l'ordre :

# pipeline complet pyperun flow my-experiment # sur une fenêtre temporelle pyperun flow my-experiment --from 2026-01-01T00:00:00Z --to 2026-02-01T00:00:00Z # tout effacer et reprendre de zéro pyperun flow my-experiment --output-mode reset

💡 Pas encore de données ? Sème le dataset de référence DEMO et teste le pipeline de bout en bout : python scripts/seed_demo.py puis pyperun flow demo.

5

Vérifier le statut

Voir l'état à jour / en retard de chaque dataset :

pyperun status

Lister ce qui est disponible :

pyperun list flows pyperun list treatments pyperun list steps --flow my-experiment
6

Rejouer une seule étape

Pas besoin de tout relancer. Cible une étape précise :

# une seule étape pyperun flow my-experiment --step clean # d'une étape jusqu'à la fin pyperun flow my-experiment --from-step resample
7

Comprendre le pipeline

10 traitements enchaînés, chacun lit un dossier et écrit le suivant :

#TraitementEntrée → SortieRôle
1parse00_raw → 10_parsedCSV clé:valeur → parquet typé
2clean10_parsed → 20_cleandédoublonne, bornes, anti-spikes
3resample20_clean → 25_resampledgrille régulière 1 s, ffill
4transform25_resampled → 30_transformmaths déclaratives (log, cbrt_inv)
5normalize30_transform → 35_normalizednormalisation par percentile
6aggregate35_normalized → 40_aggregatedfenêtres 1s/10s/60s/5min/1h
7to_postgres40_aggregated → PostgreSQLexterne tables Grafana
8exportcsv40_aggregated → 61_exportcsvCSV par device
9exportparquet40_aggregated → 62_exportparquetparquet unique par device
10exportduckdb40_aggregated → 63_exportduckdbbase DuckDB unique
8

Et après ?

  • → Configure chaque étape via pyperun/treatments/<nom>/treatment.json ou --params '{…}'.
  • → Édite ton pipeline dans flows/my-experiment.json (ajoute/retire des étapes).
  • → Lance les tests : python -m pytest tests/ -v.
  • → Pilote tout depuis l'interface graphique.
